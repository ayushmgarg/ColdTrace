Distributed Computing Concepts — Demonstrated in this System
Hybrid Architecture
IGD stores locally when cloud is offline; cloud is authoritative source of truth.
Client-Server Hierarchy
LTAT → IGD → Cloud → Dashboard. Each layer is client to the next.
Gateway Types
IGD (storage hub), CCVE (vehicle + GPS/Beidou), CCIE (portable incubator) — all with store-and-forward.
Fault Tolerance
Store-and-forward SQLite buffer in gateway. 5 redundant TMP112 sensors per LTAT device.
Message Persistence
Gateway buffers packets during outages; flush-on-reconnect restores data to cloud without loss.
Alert Engine (24/7)
Runs at cloud level — guarantees alerts even without active user. Rolling avg + linear regression detect excursions.
Reliable Multicast
Alert engine broadcasts to all registered managers simultaneously via SMS (Twilio) + email (SendGrid).
Clock Synchronisation
UTC timestamps on every packet. Preserved from acquisition → gateway → cloud.
